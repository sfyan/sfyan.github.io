React Native版Booking APP出炉

发表于 2020-01-07 更新于 2020-02-27 分类于 原创 ， react ， react-native

宅家数周，很适合思考人生，更适合撸代码。
最近react native技痒，不知不觉的就把Booking APP的主要UI及功能给重制出来了。

Booking APP虽产品成熟但年代久远，很多底层代码都难以更新。产品和开发每天在做的也只能是小范围的AB试验、优化和修补。
所以不难想到，如果用新框架做出Booking APP的话，可以省去多少的维护成本，增添多少新的可能性。
前提当然是新框架可以实现目前APP的所有功能。于是有了这一次尝试，看看RN是否能实现重要的Booking APP功能。

目前是v1 demo版，已实现的功能有：
首页, 轮播图, 搜索框（含成员设置功能）, 头像（含拍照和图库功能）, 搜索结果页, 结果各类排序.

作为一个前设计师，我感觉UI还原是没什么问题了。这里还要感谢一下大设计师Lulu，为APP提供了连Booking主APP都没有的loading动画。
但是功能上肯定还是有很多尚未完成和需要提高的地方，希望在接下来的版本可以改进。目前安卓APP已经全面测试，可以移步GitHub 获取APK。IOS版也随时可以从RN项目导出，由于尚未测试就暂时不公布了。

主要Components

Home.js : TopBar.js``SearchBox.js``DiscountSection.js``Menu.js``Setting.js
SearchResult.js:FilterBar.js``FilterMenu.js``SRTopBar.js``HotelBox.js

npm install -S -D -g的区别

发表于 2019-11-24 分类于 转帖 ， npm

作者：itxing
链接：https://www.jianshu.com/p/2e7f3b69e51e
来源：简书

npm install module_name -S 即 npm install module_name --save 写入dependencies

npm install module_name -D 即 npm install module_name --save-dev 写入devDependencies

npm install module_name -g 全局安装(命令行使用)

npm install module_name 本地安装(将安装包放在 ./node_modules 下)

dependencies与devDependencies有什么区别呢？

devDependencies里面的插件只用于开发环境，不用于生产环境

dependencies是需要发布到生产环境的

举个例子：

你开发一个前端项目，在项目中你需要使用gulp构建你的开发和本地运行环境,这时你就要放到dependencies里。gulp是你用来压缩代码，打包等需要的工具，程序实际运行的时候并不需要，所以放到dev里就ok了。

你写程序要用element-ui,生产环境运行项目时肯定要用到element-ui,这时element-ui就应该安装到dependencies中去。

webpack的css loader、url loader

发表于 2019-11-10 更新于 2019-11-24 分类于 原创 ， wepack

JS文件中import ‘./css/index.css' 无法直接生效，是由于：Webpack 只能打包处理JS类型的文件，无法处理其他非JS文件。针对非JS类型的文件的处理，需要手动安装第三方loader加载器。

处理css文件步骤如下：

  1. 安装 npm i style-loader css-loader less-loader sass-loader url-loader file-loader -D

(sass-loader需要安装node-sass npm i node-sass -D，url-loader需要安装file-loader）

  1. 打开配置文件webpack.config.js，新增配置节点module(对象），在这个module对象内部，有个rules属性（数组），该属性存放了所有第三方文件的匹配和处理规则。
  1. 调用规则：从右到左

{"dev":webpack-dev-server

Hexo Next自定义目录

发表于 2019-10-31 更新于 2019-11-24 分类于 原创 ， hexo

Hello World!

第一篇日志，
记录一下Hexo Next的目录自定义方法：

1./myblog/themes/next/_config.yml
去除#号开启默认目录，新增目录则
目录名key: /目录名/ || icon名

# ---------------------------------------------------------------
# Menu Settings
# ---------------------------------------------------------------

# Usage: `Key: /link/ || icon`
# Key is the name of menu item. If the translation for this item is available, the translated text will be loaded, otherwise the Key name will be used. Key is case-senstive.
# Value before `||` delimiter is the target link, value after `||` delimiter is the name of Font Awesome icon.
# When running the site in a subdirectory (e.g. domain.tld/blog), remove the leading slash from link value (/archives -> archives).
# External url should start with http:// or https://
menu:
  home: / || home
  about: /about/ || user
  tags: /tags/ || tags
  categories: /categories/ || th
  archives: /archives/ || archive
  download: /download/ || download
  #schedule: /schedule/ || calendar

2./myblog/themes/next/languages/zh-CN.yml
可添加目录对应的中文翻译
